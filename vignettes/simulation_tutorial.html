Module Perturbation on a Simulated Branching Trajectory

Compiled: 2026-05-04

Source: vignettes/simulation_tutorial.Rmd

Introduction

In this tutorial, we demonstrate compact using a simulated single-cell dataset with a known branching trajectory. Simulation offers several advantages:

  1. Ground truth: We know exactly which genes drive differentiation along each branch, so we can evaluate whether perturbation results are biologically sensible.
  2. Reproducibility: The data is generated deterministically (with a fixed random seed), so results are fully reproducible and anyone can regenerate the dataset.
  3. Accessibility: No data-sharing restrictions — you can easily share the code with collaborators.

This tutorial builds on hdWGCNA, our previous package for weighted gene co-expression network analysis in single-cell data. If you are unfamiliar with co-expression modules or need to run hdWGCNA yourself, we recommend starting with the hdWGCNA tutorial. The data for this vignette has already been processed with Seurat, hdWGCNA, and monocle3; this tutorial focuses exclusively on the perturbation step.

The Simulated Dataset

The dataset contains ~4,800 cells and ~6,000 genes, simulated with splatter to recapitulate a branching developmental trajectory with three main branches:

  • Branch 1 (orange): Shared root → Tip A, divided into four pseudotime groups (B1-1 → B1-4)
  • Branch 2 (green): Shared root → Tip B, divided into four pseudotime groups (B2-1 → B2-4)
  • Branch 3 (purple): Shared root → Tip C, divided into four pseudotime groups (B3-1 → B3-4)

hdWGCNA has identified three co-expression modules: turquoise, blue, and yellow. Each module has a distinct expression pattern across the branches, reflecting the biological drivers of differentiation.

In this tutorial, we perform an in-silico perturbation of the turquoise module in both directions (knock-down and knock-in) and observe how the inferred cell-state transitions change. Since we know turquoise is preferentially active along a specific branch, we expect: - Down-regulating turquoise → cells should transition away from the turquoise-enriched branch - Up-regulating turquoise → cells should transition toward the turquoise-enriched branch

Let’s verify this intuition using compact’s tools.

# Section 2: Building the KNN Graph

Before running `ModulePerturbation`, we must construct a nearest-neighbor graph. This graph defines the **local neighborhoods** over which `compact` computes cell-cell transition probabilities. Essentially, transition probabilities are only computed between neighboring cells in the graph.

Run `FindNeighbors` to build the SNN (shared nearest neighbor) graph:

```{r eval=FALSE}
seurat_obj <- FindNeighbors(
    seurat_obj,
    reduction = 'pca',
    assay = 'RNA',
    k.param = 20,
    annoy.metric = 'cosine'
)

Parameters:

  • k.param = 20: Each cell connects to its 20 nearest neighbors in PCA space. For datasets with 1,000s of cells, values between 15 and 50 typically work well; for smaller datasets, lower values are appropriate.
  • annoy.metric = 'cosine': Use cosine distance (standard for high-dimensional gene expression data). Alternative: 'euclidean'.
  • reduction = 'pca': Compute neighbors in PCA space rather than the full gene expression matrix (more efficient and robust).

The SNN graph is stored in the Seurat object at seurat_obj@graphs[['RNA_snn']]. This is the graph ModulePerturbation will use to compute transition probabilities in the next section.

Note: The graph is only computed once, even if you perform multiple perturbations. You can run ModulePerturbation sequentially for different modules or different perturbation directions without re-running FindNeighbors.
